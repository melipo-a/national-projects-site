# Инструкции для Codex

## Главная задача

Нужно создать новый учебный многостраничный сайт на русском языке по проекту «Национальные проекты России».

Сайт должен быть сделан заново, но содержание нужно брать из файла:

project-materials/source-text.md

Нельзя придумывать основную идею проекта с нуля. Нужно опираться на исходный исследовательский текст, цель проекта, требования преподавателя и структуру, указанную в source-text.md.

## Самое важное по замечаниям преподавателя

1. Нельзя механически делать сайт как 7 отдельных тем по числу участников.
2. Семь исследовательских частей должны быть встроены в сайт через смысловые разделы, карточки, ссылки, схемы, данные и визуализации.
3. Цель проекта должна быть единой для всей команды.
4. Исследование должно быть не только социологическим по ВЦИОМ, но и практическим:
   - официальные сайты национальных проектов;
   - сайты исполнения бюджетов;
   - данные о реализации самих проектов;
   - практические результаты.
5. Исследования и выводы нужно показывать через диаграммы, графики, карточки, схемы и понятные визуальные блоки.
6. Научность проекта нужно показать через подборку научных статей.
7. Сайт не должен выглядеть как презентация или набор случайных блоков.

## Технический стек

Использовать только:
- HTML
- CSS
- JavaScript

Не использовать:
- React
- Vue
- Angular
- Next.js
- сборщики
- npm-зависимости
- внешние UI-библиотеки

Сайт должен открываться напрямую через GitHub Pages.

## Требования к сайту

- Многостраничная структура.
- Адаптивная верстка для компьютера и телефона.
- Современный, аккуратный, учебный, но не скучный дизайн.
- Никакие блоки не должны съезжать.
- Навигация должна работать через относительные ссылки.
- Тексты должны быть на русском языке.
- Нельзя делать сайт пустым.
- Нельзя копировать длинные куски текста из source-text.md без переработки под сайт.
- Нельзя вставлять бессмысленные декоративные картинки.
- Иллюстрации, карточки, схемы и графики должны помогать понять тему.
- Нужно предусмотреть страницу или блок с источниками.

## Редакционный принцип

Исходный файл составлен из 7 частей, но сайт не должен повторять эту структуру буквально.

Лучше строить сайт вокруг логики:

1. Что такое национальные проекты.
2. Как они связаны с национальными целями.
3. Как устроена цепочка от цели к результату.
4. Как проекты влияют на жизнь человека.
5. Как проекты меняют среду, инфраструктуру и регионы.
6. Как экономика, цифровизация и технологии связаны с развитием страны.
7. Что показывают данные ВЦИОМ и практические источники.
8. Какие научные статьи подтверждают исследовательскую основу проекта.
9. Где пользователь может проверить источники.

## Желательная структура сайта

Можно предложить и улучшить структуру, но базовая логика такая:

- Главная страница: объяснение идеи сайта и цепочки «цель → проект → бюджет → реализация → результат».
- Страница «Как устроена система»: национальные цели, нацпроекты, федеральные проекты, инициативы, региональная реализация.
- Страница «Для человека»: семья, молодежь, образование, кадры, здоровье.
- Страница «Среда и регионы»: инфраструктура, экология, транспорт, туризм.
- Страница «Экономика и технологии»: экономика, экспорт, цифровизация, технологическое лидерство.
- Страница «Данные и восприятие»: ВЦИОМ, узнаваемость проектов, коммуникационные проблемы, почему информированность не равна пониманию.
- Страница «Источники и научная база»: официальные источники, бюджетные данные, научные статьи.

## Визуализации

На сайте желательно сделать:
- схему «национальная цель → национальный проект → федеральные проекты → меры → региональная реализация → результат»;
- карточки смысловых блоков;
- диаграммы по данным ВЦИОМ;
- блок с бюджетной логикой и финансированием;
- блок «официальное название / понятное название»;
- карту или условную схему направлений;
- блоки с практическими примерами результатов;
- страницу источников с разделением на официальные, социологические и научные.

## Стиль текста

Текст должен быть:
- понятным;
- учебным;
- человеческим;
- без чрезмерно нейросетевых формулировок;
- без лишней воды;
- не слишком официальным;
- достаточно серьезным для учебного проекта.

## Проверка перед завершением

Перед финальным ответом нужно проверить:
- все ли страницы открываются;
- все ли ссылки работают;
- нет ли съехавших блоков;
- корректна ли адаптация под мобильный экран;
- нет ли пустых разделов;
- нет ли повторяющихся бессмысленных изображений;
- сайт готов для GitHub Pages;
- замечания преподавателя учтены;
- сайт не повторяет структуру «7 отдельных тем» буквально.
